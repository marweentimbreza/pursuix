This is the first note on a site I'll keep adding to as I learn the defensive side of security. It's deliberately plain: static files, no tracker, no third-party scripts on the page you're reading. If you're here from a resume link — welcome. The writing is the proof.

How this gets written

Two ways, both ending in the same clean HTML. I either type straight into a browser editor — select text and a small toolbar appears for headings, links, and code — or I drag a LibreOffice/Word document onto the page and it's converted in place. Nothing about the draft leaves my machine until I choose to publish.

Why static, why no analytics

Every script on a page is something a reader has to trust. I'd rather hold none of that. Posts are just files in a git repo; the site is the repo, served as-is. The trade is that publishing is a commit instead of a button — which suits how I already work.

  • Source of truth: a small posts.json the homepage reads to build this list.
  • Each post: its own file under /posts/, pulled in when you open it.

What's coming

Notes on reading network traffic, triaging alerts, and turning what I find into detections — written as I go, mistakes left in where they're useful. If something here is wrong or you want to compare notes, the contact form on the about page reaches me.
